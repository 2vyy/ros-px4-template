## Why this matters

Three declaration-level inconsistencies found by a toolchain audit:

1. `lark` is imported NOWHERE in the repo but is declared in BOTH the
   pyproject dev group and tasks.py's PEP 723 inline block, and resolved into
   `uv.lock`. Dead weight in every `uv sync` and script-env build.
2. `tomli>=2.0.0` in the inline block is dead: Python 3.12 has `tomllib`, and
   the only TOML writer is `tomli_w` (`tools/capabilities.py:10`).
3. The two dependency lists (pyproject `[project]` vs the inline block) are
   INTENTIONALLY different - the inline block gives the task runner a minimal
   isolated script env, which is a good and subtle design - but nothing says
   so, and they are already drifting (`typer>=0.12.3` inline vs `>=0.12` in
   pyproject). A contributor will eventually "fix" one to match the other.

Bundled (same hygiene pass, one-line diff): the `justfile`'s `scenario`
recipe takes exactly one argument while every sibling forwards `*args`, so
`just scenario 01_arm_takeoff --anything` is impossible even when
`tasks.py scenario` grows options.

## Steps

### Step 1: Re-verify the deps are dead

```
rg -n "lark" --glob '*.py' --glob '!plans'
rg -n "^import tomli$|^from tomli import|import tomli\." --glob '*.py'
```

Expected: the first matches only tasks.py's inline-comment line; the second
matches nothing (`tomli_w` is a different package and stays).

**Verify**: as above. Any real import is a STOP condition.

### Step 2: Edit the two dependency declarations

1. `pyproject.toml`: delete the `"lark>=1.1.9",` line from
   `[dependency-groups] dev`.
2. `tasks.py` inline block: delete the `tomli>=2.0.0` and `lark>=1.1.9`
   lines; change `"typer>=0.12.3"` to `"typer>=0.12"` (match pyproject).
3. Add a comment line inside EACH list pointing at the other. In the inline
   block, after the `dependencies = [` line:
   `#     # Minimal isolated env for the task runner (uv script mode) - intentionally`
   `#     # smaller than pyproject [project]; keep floors aligned when both declare a pkg.`
   And in `pyproject.toml`, above `[dependency-groups]` or next to the
   `[project] dependencies`: a one-line
   `# tasks.py runs in uv script mode with its own smaller inline dep list (see tasks.py:1-12).`

4. `uv lock` then `uv sync`.

**Verify**: `rg -n "lark" uv.lock` -> no matches;
`uv run tasks.py --help` -> prints command help (script env rebuilt fine);
`uv run pytest tests/unit -q` -> no new failures.

### Step 3: Fix the `scenario` recipe

Replace:

```make
scenario name:
    @just _run scenario "{{name}}"
```

with the sibling pattern:

```make
scenario *args:
    @just _run scenario "$@"
```

Keep the recipe's doc comment line unchanged.

**Verify**: `just scenario` with no args -> tasks.py's own usage error (exit
2 class), not a just parse error; `just scenario 02_hover_hold` on a
sim-less host -> the normal precondition/FAIL verdict path (proves the name
still arrives as arg 1).

### Step 4: Full gate

**Verify**: `just check` -> exit 0.
