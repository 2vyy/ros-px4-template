## What the audit found useful (keep — no plan needed)

The runtime code is lean and the *wired* tooling earns its place; these were
verified and deliberately left alone:

- English verdicts + fixed exit codes (`tools/cli_verdict.py`).
- Readiness gating + cold teardown (`tools/wait_ready.py`, `tools/preflight.py`,
  `tools/sim_cleanup.py`).
- Single logfmt `logs/latest.log` (rg-able, no `jq`).
- `tests/capabilities.toml` + `cap mark` — load-bearing: `scenario_sim_configs()`
  drives e2e group scheduling (`tasks.py:734`), not just a registry.
- `tools/status.py`, `tools/e2e_report.py`, the detached-sim contract.

  build it.
